Data Menu

The Data menu allows selection of the displayed data type. In image windows, the value of the displayed data maps directly into the color scale defined by the Event Histogram. The various items of the Data menu are described below.

Uncalibrated (calibrated and fitter versions only)

Displays RAW ADC data values for Tac, Qhs, Qhl and Qlx, or the raw hits for Hit Count display.
Calibrated (calibrated and fitter versions only)
Displays calibrated time and charge values for each hit using simplified (linear) calibration constants. Currently, time values are displayed in nanoseconds, and charge values pedestal corrected but are still displayed in raw ADC counts.
Cal No Walk (calibrated and fitter versions only)
Same as Calibrated, but with no time walk (Q vs. T) correction. The time walk correction uses the value of Qhs to correct for the time walk of the discriminator. Note that the correction effects the time offsets, so fits will be affected. Turning on the time walk correction slows down loading of events by a factor of two when summing calibrated hits for optical calibration (since this operation imposes a time window on each hit before summing).
SNODB Calibration (ROOT_FILE version only)
Applies calibration by loading SNODB constants from a network server and calculating calibrated values using the QSnoCal ROOT object.
SNOMAN Calibration (ROOT_FILE version only)
Applies a full SNOMAN calibration by sending each event to a SNOMAN calibrator network server.
Pre-calibrated (ROOT_FILE version only)
Used for displaying data which has already been calibrated. Will display existing calibrated values for Tac, Qhs, Qhl and Qlx if available. Note that pre-calibrated data is only available when reading from a file, and currently only ROOT files contain calibrated data.
Tac
Sets the color and histogram scales to display raw TAC values for each hit, or calibrated TAC values if calibrated data is displayed. See the section on the Hit Info window for more detail about the Tac values.
Time Diff (calibrated and fitter versions only)
Displays the time difference between the straight-line fitted time and the actual measured time for each hit. The speed of light used for this calculation is obtained from the value of the light_speed resource.
Qhs
Displays the high gain, short integrate ADC value (pedestal corrected if calibrated data is displayed).
Qhl
Displays the high gain, long integrate ADC value (pedestal corrected if calibrated data is displayed). See the section on the Hit Info window for more detail about the Qhl values.
Qlx
Displays the long integrate (high or low gain, depending on the hardware settings) ADC value (pedestal corrected if calibrated data is displayed). See the section on the Hit Info window for more detail about the Qlx values.
Qhl-Qhs
Displays the difference between high gain long and short integrate (pedestal corrected if calibrated data is displayed).
Hit Count
Displays the number of times each tube was hit. For normal events, each participating tube should be hit exactly once. This display is useful mainly when displaying summed events. Note: The histogram produced in this mode is NOT like a standard NHIT histogram. It is a histogram of the number of times each tube was hit (as opposed to the number of tubes hit in each event). See the section on the Hit Info window for more detail about the Hit Cnt values.
NCD Shaper Value
Displays the NCD shaper value as the color for each NCD hit. In sum mode, this is the average shaper value.
NCD Shaper Hits
Displays the number of times the NCD Shaper fired as the color for each hit.
NCD MUX Hits
Displays the number of times the NCD MUX fired as the color for each hit.
NCD Scope Hits
Displays the number of times the NCD Scope fired as the color for each hit. The scope data must exist or the hit doesn't contribute to the scope count.
Crate
Displays the crate ID number as the color for each hit.
Card
Displays the FEC card number as the color for each hit.
Channel
Displays the FEC channel number as the color for each hit.
Cell
Displays the CMOS cell number as the color for each hit.
CMOS Rates (dispatcher version only)
Displays the CMOS rates information. This special feature is only useful for displaying noise rates when connected to a dispatcher for online monitoring. CMOS rates are not normally stored to event files.
