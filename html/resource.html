XSNOED Resource File

The first time you run XSNOED, the default resource file "XSnoed" is copied from the XSNOED directory to a settings file called ".XSnoed" in your home directory (unless you specified a different settings file name through the "-s" command line option). This file is used to specify all XSNOED resources such as the colors of XSNOED windows and menus, as well as to save the current XSNOED settings. These settings are loaded automatically the next time you run XSNOED.

The resource section of the settings file can be edited with any word processor to change some basic XSNOED parameters. The list below explains the particular XSNOED resources that can be changed in this way.

Note that any resource specified here can also be set on the command line when running XSNOED. For instance, to change the water level to 300 cm, run XSNOED with the command line "xsnoed -water_level 300".

When writing settings to the file, XSNOED preserves existing resources, and only overwrites an automatically generated section (not shown here) at the end of the settings file.

Note that the alternate colors defined below specify settings for the White Background color set as selected in the Display menu or printer settings.

Notes:

[1] This water level is ignored by the ROOT_FILE version which gets its water level information from the ROOT QWater object.

[2] Simplified linear calibration constants only.

[3] The calibrated charge values are expressed in channels. For PE (photoelectron) calibrated charge scales, these values are divided by 20 for Qhs (and Qhl-Qhs), 40 for Qhl, and 2 for Qlx.

[4] Used by the OPTICAL_CAL version only.

[5] Used only by XSNOMAN and ROOT versions when extra hit data is available.

[6] Used by the SNOPLUS version only.

Note: Occasionally at the release of a new version of XSNOED, the version of the resource file will also be changed. If this happens, XSNOED will print a message at startup indicating that your settings file is out of date. Then your original settings file will be copied to a backup file (of the same name with a "~" added to the end), and a new settings file will be created. The new settings file will consist of the resources from the default XSnoed resource file and the settings from your original settings file. At this point, any resources that you changed manually in the original settings file will have to be re-entered in the new settings file.
