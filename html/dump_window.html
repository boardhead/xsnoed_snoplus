Dump Data Window

The Dump Data window allows you to dump ASCII-formatted PMT data from the currently displayed event (or sum of events) to file.

The Filename field is used to enter the full pathname of the output file. If the specified file already exists, output will be appended to the file if Append is selected, or will overwrite the old file if Overwrite is selected. The filename field may be left blank, in which case output is written to the console.

The Comment field is provided to allow additional comments about the event to be saved. After the comment is written to file, this text field is cleared to allow input of the next comment. Previous comments can be recalled by scrolling through a history buffer using the Up and Down keys on the keyboard. Note that the comment history is not saved like the filename when this window is closed. Also note that duplicate entries are removed from the history to reduce the amount of scrolling required to find the desired entry.

Pressing the Write button causes the event information, comments and PMT data to be written to the specified file. Alternatively, you can press the Return key when editing any of the text fields to perform the same function.

Pressing Done closes the window.

Below is a sample output from one event. The comment, GTID, number of hits, and data type are written to the first lines of the file (these lines all begin with "#" to allow them to be skipped easily if this file is processed later). The GTID is always written in decimal, regardless of the current settings. The remaining lines give the crate, card, channel and data value for each PMT currently displayed. The data type is the same as currently displayed, and is selected through the Data menu.
